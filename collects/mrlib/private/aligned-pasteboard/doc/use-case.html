on-size method of the vertical-pasteboard% and horizontal-pasteboard% classes

there is no on-size event of the pasteboard% class. it is not notified when it's display gets an on-size event. this causes a behavoir that is unlike panels which get an on-size event when they are resized. in order to get the vertical-pasteboard% objects to behave more like vertical-panel% objects i will derive editor-canvas% and editor-snip% classes for the purpose of displaying vertical-pasteboard% objects.

the editor-canvas% on-size method will be overriden to call send an on-size message to it's vertical-pasteboard% editor.

Title
on-size of vertical-pasteboard%
Actors
user, frame%, aligned-editor-canvas%, vertical-pasteboard%, snip%
use case
user frame% aligned-editor-canvas% vertical-pasteboard% snip%
  • user resizes the window
  • frame gets and on-size method changes its size
  • sends on-size messages to its child aligned-pasteboard-canvas%
  • canvas gets on-size and changes its size
  • sends on-size to its vertical-pastaboard% child
  • pasteboard gets on-size and chages its size.
  • pasteboard sends get-min-height to all stretchable snips
  • pasteboard gets get of all unstretchable snips (with get-snip-location method)
  • stretchable snip% gets get-min-height and returns its minimum height
  • pasteboard adds the minimum heights of the stretchable snips to the hieghts of the others
  • pasteboard calculates how much size is left over and divided by the number of sizeable snips
  • pasteboard sends the size-left over per sizable value to the resize method of each sizeable snip
  • resizable snips change their size
