PLT Web Server Embedding Interface

To use the Web server inside another PLT-Scheme program, require the Web server module:

(require (lib "web-server.ss" "web-server"))
This provides one main function:
serve : [nat] [nat] [(str -> str)] -> (-> void)
(define (serve port max-wait virtual-hosts) ...)
It starts the server, where port allows the server to listen on a port besides 80 and max-wait lets the caller specify the maximum number of clients that can wait for their TCP connection to be accepted. The virtual-hosts function maps host names, which may or may not include a colon followed by a port number, to a directory name. The return value is a thunk that shuts down the server. The web server serves files out of the directory (build-path (collection-path "web-server") "default-web-root" "htdocs"), unless the function serve was called with a virtual-hosts argument. In that case, files are served out of the htdocs directory under the directory returned by the virtual-host function.
